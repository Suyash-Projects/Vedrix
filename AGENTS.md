# Vedrix AI Interview System — Agent Instructions

## Project Identity

**Name**: Vedrix AI Interview System  
**Stack**: FastAPI + React + SQLite + LangGraph  
**Goal**: High-fidelity, agentic AI interview platform with LangGraph orchestration.
**Dual-sided**: B2C (candidates) + B2B (recruiters)

## Quick Start

```bash
python Vedrix/run_dev.py
```

- Auto-allocates free ports for backend (8000+) and frontend (5173+)
- Starts both services concurrently
- Writes `VITE_API_URL` to `.env.development.local`
- Check console output for actual URLs

## Environment Setup

| File | Purpose |
|------|---------|
| `Vedrix/backend/.env` | Backend config (pydantic-settings loads automatically) |
| `Vedrix/frontend/.env.development.local` | Frontend API URL (generated by run_dev.py) |

**Database**: SQLite (`sqlite+aiosqlite://`) by default  
**PostgreSQL**: Run `docker-compose -f Vedrix/docker-compose.yml up` in Vedrix/

## Architecture

- **API prefix**: `/api/v1`
- **Endpoints**: `auth`, `users`, `profiles`, `interview`, `admin`, `hr`, `student`
- **Interview engine**: LangGraph-based (`services/interview_engine/`)
- **Frontend state**: Zustand + React Query
- **Styling**: Tailwind CSS v4 (`@tailwindcss/vite`)

## Development Commands

| Command | Purpose |
|---------|---------|
| `python Vedrix/run_dev.py` | Full stack dev (both services) |
| `cd Vedrix/backend && python -m pytest` | Run backend tests |
| `cd Vedrix/frontend && npm run lint` | ESLint check |
| `cd Vedrix/frontend && npm run build` | Production build |
| `docker-compose -f Vedrix/docker-compose.yml up` | PostgreSQL + Redis |

## Tech Constraints

- **Async-first**: All backend I/O (DB, AI APIs) must be `async`
- **Database URL**: Must use `sqlite+aiosqlite://` — config.py auto-fixes if you forget
- **Rate limiting**: Enabled via slowapi (see `app/core/rate_limit.py`)

## AI Services

- **Groq**: Low-latency interview loop
- **DeepSeek / NVIDIA**: Deep analysis and feedback
- **OpenRouter**: Unified API gateway
- Keys are in `Vedrix/backend/.env`

## Gotchas

1. Frontend won't connect to API without `.env.development.local` — always run via `run_dev.py` or create manually
2. SQLite DB file: `Vedrix/backend/vedrix.db` (check .gitignore)
3. Tailwind v4 uses `@tailwindcss/vite` plugin, not postcss
