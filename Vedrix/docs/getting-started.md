# Getting Started with Vedrix

Welcome to Vedrix! This guide will help you get the system up and running on your local machine.

## System Requirements

Before you begin, ensure you have the following installed:

- **Git** - For version control
- **Python 3.12+** - Backend runtime
- **Node.js 20+** - Frontend runtime
- **Docker & Docker Compose** - For database and caching services
- **Git** - For cloning the repository

## Installation

### 1. Clone the Repository

```bash
git clone <repository-url>
cd Vedrix
```

### 2. Environment Setup

The system uses environment variables for configuration. For development, most variables have sensible defaults.

**Backend Configuration** (`Vedrix/backend/.env`):
```bash
# Optional: Create backend environment file
# cp .env.example .env  # If example exists
# Edit as needed
```

**Frontend Configuration**:
The frontend configuration is automatically generated by the development script.

### 3. Start Development Services

```bash
# Start both backend and frontend services
python run_dev.py
```

The script will:
1. Check for required dependencies
2. Auto-allocate ports to avoid conflicts
3. Start the backend server on port 8000+
4. Start the frontend development server on port 5173+
5. Generate the frontend environment file with the correct API URL
6. Display the actual URLs in the console

### 4. Verify Installation

Once both services are running, you should see output similar to:

```
Backend running on: http://localhost:8001
Frontend running on: http://localhost:5173
API Documentation: http://localhost:8001/docs
```

Open your browser and navigate to the frontend URL to access the Vedrix application.

## Troubleshooting

### Common Issues

**Port Already in Use**
- The `run_dev.py` script automatically finds free ports
- If you encounter issues, check the console output for the actual ports being used

**Database Connection Errors**
- Ensure Docker is running: `docker info`
- Start required services: `docker-compose -f Vedrix/docker-compose.yml up -d`
- Wait a few seconds for the database to initialize

**Frontend Cannot Connect to API**
- Verify the frontend `.env.development.local` file contains the correct backend URL
- Check that the backend server is running and accessible

**AI Service Errors**
- Ensure API keys are configured in `backend/.env`:
  - `GROQ_API_KEY`
  - `OPENROUTER_API_KEY`
  - `DEEPSEEK_API_KEY` (if using)

### Checking Service Status

```bash
# Check if backend is responding
curl http://localhost:8000/health

# Check if frontend is accessible
curl http://localhost:5173

# View backend logs
docker-compose logs -f backend

# View frontend logs  
docker-compose logs -f frontend
```
