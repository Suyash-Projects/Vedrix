# 🚀 Implementation Plan: Humanizing Vedrix AI

This plan outlines the technical roadmap for transforming the Vedrix AI interview platform from a turn-based, manual-click system into a natural, human-like conversational experience using a **$0 cost** infrastructure.

---

## 1. Objective
Achieve "Human-in-the-Loop" feeling with:
- **Zero-Click Interaction:** Continuous speech detection.
- **Natural Voice:** Neural TTS that sounds human, not robotic.
- **Full Duplex:** Support for user interruptions ("Barge-in").
- **Presence-Based UI:** Visual feedback that reacts in real-time to audio states.

---

## 2. Technical Architecture Changes

### A. Frontend: Continuous Voice Activity Detection (VAD)
**Goal:** Remove the "Record" button and auto-detect when the user starts/stops speaking.
- **Library:** `@ricky0123/vad-react` (Silero VAD via WASM).
- **Files to Modify:** `Vedrix/frontend/src/pages/InterviewRoom.jsx`.
- **Logic:**
  - Initialize `useMicVAD` hook on component mount.
  - `onSpeechStart`: Immediately pause/mute any active AI audio playback.
  - `onSpeechEnd`: Convert `Float32Array` to WAV and send over WebSocket.
  - Remove `toggleRecording` manual logic.

### B. Backend: High-Fidelity Zero-Cost TTS
**Goal:** Replace `window.speechSynthesis` with human-like neural voices.
- **Choice:** `Edge-TTS` (Unofficial Microsoft Edge API) for best quality or `Kokoro-82M` for local privacy.
- **Files to Modify:** `Vedrix/backend/app/services/voice_service.py` (to be created/extended).
- **Logic:**
  - Implement a streaming wrapper for TTS.
  - Send audio chunks over WebSocket as they are generated by the LLM.
  - Use `Edge-TTS` to generate MP3/Opus buffers.

### C. Backend: Truncation & Interruptions
**Goal:** Stop the AI "brain" immediately when the user starts speaking.
- **Files to Modify:** `Vedrix/backend/app/api/v1/endpoints/interview.py` (WS handler).
- **Logic:**
  - Add a new WS message type: `{"type": "interrupt"}`.
  - When received, use a `CancellationToken` (or flag in LangGraph state) to abort the current generation.
  - Clear the audio queue on the frontend.

### D. UI/UX: The "Presence" Orb
**Goal:** Move away from bar graphs to an abstract, responsive avatar.
- **Component:** New `AIOrb.jsx` (replacing `Waveform`).
- **States:**
  - **Idle:** Slow, deep purple breathing.
  - **Listening:** Blue-cyan pulse, reactive to volume.
  - **Thinking:** Rapidly shifting gradients/morphing shape.
  - **Speaking:** Expanding purple-gold glow synced with audio frequency.
- **Live Transcription:** Overlay a "Ghost Text" box that shows real-time STT feedback to the user.

---

## 3. Implementation Phases

### Phase 1: The "Listening" Engine (High Priority)
- Install `@ricky0123/vad-react`.
- Update `InterviewRoom.jsx` to use the VAD hook.
- Implement the "Local Mute" logic on speech start.

### Phase 2: The "Voice" Engine
- Integrate `Edge-TTS` into the FastAPI backend.
- Update WebSocket handlers to stream audio bytes.
- Refactor frontend to play binary audio buffers instead of triggering browser TTS.

### Phase 3: The "Presence" UI
- Build the `AIOrb` component using Framer Motion.
- Connect Orb states to the VAD and WebSocket status.
- Implement the "Ghost Text" transcription overlay.

---

## 4. Verification & Testing
- **Latency Test:** Time-to-First-Byte (TTFB) for audio should be < 1.5s.
- **VAD Accuracy:** Test in noisy environments to ensure the "redemption period" prevents cutting off sentences.
- **Interruption Test:** Verify that speaking "Stop" or any answer during AI playback instantly halts the voice.
