## Project Overview

**Vedrix** is an AI-powered interview platform with dual-sided access:
- **B2C (Candidates):** Practice adaptive AI interviews with real-time feedback
- **B2B (Recruiters):** Create job drives and use AI as an unbiased screening layer

## Quick Start

```bash
python Vedrix/run_dev.py
```

This script:
- Auto-allocates free ports for backend (8000+) and frontend (5173+)
- Starts both services concurrently
- Writes `VITE_API_URL` to `.env.development.local`

## Technology Stack

| Layer | Technology |
|-------|------------|
| Backend | FastAPI (Python) |
| Frontend | React (Vite, Tailwind CSS v4) |
| Database | SQLite (default) / PostgreSQL (optional) |
| AI Integration | OpenRouter (Groq, NVIDIA, DeepSeek) |
| State Management | Zustand + React Query |
| Interview Engine | LangGraph-based |

## Development Commands

| Command | Description |
|---------|-------------|

## Architecture

### Backend (`Vedrix/backend/`)
- **API prefix**: `/api/v1`
- **Endpoints**: `auth`, `users`, `profiles`, `interview`, `admin`, `hr`, `student`, `resume`, `voice`
- **Key directories**:
  - `app/api/v1/endpoints/` — API route handlers
  - `app/models/` — SQLAlchemy ORM models
  - `app/schemas/` — Pydantic request/response schemas
  - `app/services/` — Business logic (including LangGraph interview engine)
  - `app/core/` — Config, security, rate limiting

### Frontend (`Vedrix/frontend/src/`)
- `pages/` — Route components (LandingPage, Login, Register, InterviewRoom, InterviewReport, AdminDashboard, StudentDashboard, HRDashboard)
- `components/` — Reusable UI components
- `store/` — Zustand state stores
- `services/` — API client functions
- `hooks/` — Custom React hooks
- `test/` — Frontend tests

### Interview Engine (`Vedrix/backend/app/services/interview_engine/`)
- **State**: LangGraph state management
- **Graph**: LangGraph workflow definition
- **Nodes**: Individual graph nodes
- **Providers**: AI provider integration (Groq, DeepSeek, NVIDIA)

## Environment Configuration

| File | Purpose |
|------|---------|
| `Vedrix/backend/.env` | Backend config (pydantic-settings loads automatically) |
| `Vedrix/frontend/.env.development.local` | Frontend API URL (generated by run_dev.py) |

**Database**: SQLite (`sqlite+aiosqlite://`) by default at `Vedrix/backend/vedrix.db`

## Important Constraints

1. **Async-first**: All backend I/O (DB, AI APIs) must be `async`
2. **Database URL**: Must use `sqlite+aiosqlite://` — config.py auto-fixes if you forget
3. **Rate limiting**: Enabled via slowapi (see `app/core/rate_limit.py`)
4. **Tailwind v4**: Uses `@tailwindcss/vite` plugin, not postcss
5. **Frontend API**: Won't connect without `.env.development.local` — always use `run_dev.py` or create manually

## AI Services

- **Groq**: Low-latency interview loop
- **DeepSeek / NVIDIA**: Deep analysis and feedback
- **OpenRouter**: Unified API gateway
- Keys are in `Vedrix/backend/.env`
